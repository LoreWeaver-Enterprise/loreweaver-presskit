What LoreWeaver builds and why it matters

LoreWeaver offers two products that together form a complete narrative pipeline for game development. Architect handles the authoring side. Director handles the runtime side. No other tool on the market covers both.

Architect: Narrative Authoring

What it does

Architect takes raw creative material (lore documents, world bibles, design docs, scripts) and transforms it into structured, game ready narrative data. It produces quests, dialogue trees, character profiles, location descriptions, and branching storylines that developers can export directly into their game engines.

How it works

Writers import their existing documents into Architect. The system analyzes the text and extracts narrative entities: characters, locations, items, story arcs, and their relationships. Writers review and refine every extraction. They stay in full creative control at every step. When they are satisfied, they export the structured data as XML, JSON, or CSV files ready for integration into any game engine.

Key facts

  • Supports 11 input document formats and 10 media formats
  • Extracts 13 configurable entity types (characters, locations, items, story arcs, and more)
  • Processes a full document in 5 to 7 minutes
  • Exports to XML, JSON, and CSV
  • Runs on a 7 step workflow: import, schema generation, schema editing, entity extraction, entity review, relationship linking, export
  • Supports multi user collaboration with three permission levels
  • Currently in open beta

Who it serves

Architect serves narrative designers, game writers, and worldbuilders at studios of every size. Solo developers use it to structure their lore efficiently. Large studios use it to accelerate the pipeline from raw creative material to production ready game data. The tool works equally well for RPGs, adventure games, visual novels, and any genre that relies on structured narrative content.

One line description

Architect converts raw lore and world bibles into structured, game ready narrative data using AI, with the writer in full creative control.

Director: Runtime Narrative Engine

What it does

Director generates live NPC behavior, dynamic dialogue, and adaptive plot at runtime. It runs entirely on the player's device using optimized small language models. This means zero cloud costs, zero latency from server round trips, and zero data privacy concerns for the studio or the player.

How it works

Director receives structured narrative data (from Architect or any other source) and uses it to ground every AI response in the game's actual lore. When a player interacts with an NPC, Director generates contextually appropriate dialogue, tracks the consequences of player choices, and updates the game state accordingly. A conflict resolution schema validates every state change before it reaches the game, preventing the AI from breaking game logic or contradicting established lore.

Key facts

  • Runs entirely on device with no cloud dependency
  • Achieves a 94% valid response rate across 100 real RPG scenarios
  • Delivers 86.7% intent classification accuracy across 16 intent categories
  • Supports 5 genres: Fantasy RPG, Post Apocalyptic, Western, Cosmic Horror, Sci Fi
  • Validated through a full RPG integration (Faereld: Dead End) with 21 game systems, 249 items, 8 factions, and 100+ combat templates
  • Currently in pilot programs with select studios

Who it serves

Director targets studios building games where player agency and NPC reactivity matter. RPGs, open world games, simulation games, and live service titles all benefit from NPCs that respond dynamically to player behavior. Studios that care about data privacy, offline play, or eliminating ongoing cloud costs find Director especially compelling.

One line description

Director generates live NPC behavior and adaptive dialogue at runtime, entirely on the player's device, grounded in the game's actual lore.

The Full Pipeline

Architect and Director connect directly. Structured data from Architect flows into Director's runtime engine, creating a seamless pipeline from raw creative material to live in game narrative. Studios can use either product independently, but together they deliver something no competitor offers: an end to end workflow from unstructured lore to on device, lore grounded NPC behavior.

Why on device matters: Cloud based AI creates three problems for game studios. Latency from server round trips breaks immersion. Ongoing per token costs eat into margins. Sending player data to external servers creates privacy and IP risk. Director eliminates all three by running locally.
